## 1. 資料格式

### 1.1 波段數

| | 改動前 | 改動後 |
|---|---|---|
| 模型輸入 | **11 bands**（B01/02/03/04/05/06/07/08/8A/11/12，含 3 個紅邊） | **8 bands**（B01/02/03/04/08/8A/11/12） |

### 1.2 TIF 儲存格式

| | 改動前 | 改動後 |
|---|---|---|
| 格式 | **Strip TIF**（`block_shapes=[(1, 10980)]`，整行條帶） | **COG**（Cloud Optimized GeoTIFF，tiled 256×256） |
| 讀取 256×256 patch 的實際 I/O 量 | 掃描 256 整行 ≈ 需求的 **43×** | 僅讀目標 tile，精確 I/O |
| 單波段 256×256 讀速（benchmark） | B03 strip：**397 ms** | B03 COG：**43.6 ms**（9.1× 提升） |
| 8-band VRT 一次讀速（benchmark） | strip VRT：**1600 ms** | COG VRT：**450 ms**（3.6× 提升） |
| 轉換方式 | — | `convert_to_cog.py`（in-place，1546 TIF，8 workers，234 min） |

| 60m 波段處理 | — | 每波段設 `SrcRect→DstRect`，GDAL 自動 resample 至 10m |
| 存放路徑 | `/mnt/backup/…/vrt/`（NAS） | `/home/alanyh/…/MS6_sen2like_vrt/`（本機 SSD） |
| 數量 | — | 220 個場景（1 個場景缺 B02 排除） |

### 1.4 Patch 讀取方式

| | 改動前 | 改動後 |
|---|---|---|
| 訓練讀法 | 預切實體 Patch 檔（數萬張小圖，儲存在 NAS） | **VRT 動態讀取**（零實體 Patch；`rasterio.Window` 即時裁切） |

## 2. 前處理 Pipeline 變化

```
舊版（OSDMamba/11-band）
  Stack 11-band TIF → build_vrt.py（B01 為參考，60m，有 Bug）
  → random shuffle fold split（seed=42）
  → GB1.5 patch 切割（實體檔案存 NAS）

新版（DeepLabV3+/8-band，2026-05-20 起）
  Step 0：gpkg_to_labelme.py（GPKG → JSON，ref: B02）
  Step 1：json_to_mask_tif.py（新版 mask，default=255）
  Step 2：build_vrt_ms6.py（B02 為參考，10m，本機 SSD）
           + convert_to_cog.py（1546 TIF → COG）
           + generate_nirRG_png.py（test 場景假彩色 PNG）
  Step 3：build_scene_splits_stratified.py（stratified scene-level fold split）
           2025 年全部 → test（固定），非2025 → 5-fold（按地點×年月分層）
  Step 4：generate_patch_coords.py（GB1.0 patch 座標 TXT）
  Step 5：main_runner.py + experiments_CV.yaml → DeepLabV3+ 5-fold CV
```

### Fold Split 策略對比

| | 改動前 | 改動後 |
|---|---|---|
| 分割方式 | Random shuffle（seed=42） | **Stratified scene-level**（地點×年月分組） |
| Test set | 按 fold 輪換 | **2025 年全部場景（66 個）固定為 test** |
| 分組單位 | 場景 | **（地點 × 月份）**，防止同事件被拆到 train/test 兩側 |
| 平衡目標 | 無 | Gulf/Atlantic 比例、2019/2020/2024 年份比例均勻 |

## 4. 大圖重組（Reconstruction）

### 4.1 影像讀取方式

| | 改動前 | 改動後 |
|---|---|---|
| 方法 | `rasterio.open(vrt_path).read()`（GDAL 單執行緒讀所有 band） | **`_read_vrt_parallel`**：解析 VRT XML → ThreadPoolExecutor(8) 並行讀 8 個 band TIF → `cv2.resize` 升採樣 |
| 根本瓶頸 | GDAL deflate 解壓縮：CPU 單執行緒 | 8 個 band 同時在不同 CPU core 解壓 |
| 全場景讀取時間（10980×10980） | **~166 s** | **~39 s**（4.2× 提升） |

### 4.2 推論設定

| 參數 | 改動前 | 改動後 | 效果 |
|---|---|---|---|
| `infer_batch_size` | 8 | **64** | batch overhead 減少 8×（GPU VRAM 夠用） |
| `stride` | 192 | **256** | patch 數 3249 → 1849（43% 減少，無重疊） |
| `num_workers` | 2 | **8** | preprocessing 並行度提升 |
| fp16 inference | 無 | **`torch.autocast('cuda')`** | ~1.5× 推論加速 |
| preprocessing executor | 每 batch 重建/銷毀 | **跨 batch 重用**（單一 executor 貫穿全場景） | 消除 29 次建立開銷 |

### 4.3 後處理（視覺化輸出）

| | 改動前 | 改動後 |
|---|---|---|
| bg PNG 讀取 | 推論後同步讀（NAS，40s 阻塞） | **推論前預取（prefetch）**：提交至 `_save_pool`，結果於推論結束後取用（40s 隱藏於推論中） |
| imwrite（JPG + PNG） | 同步執行（26s 阻塞） | **非同步執行**：`_save_pool`（ThreadPoolExecutor, max_workers=2），隱藏於下一場景的推論中 |

### 4.4 速度對比（單場景，10980×10980）

| 階段 | 改動前估計 | 改動後實測 | 說明 |
|---|---|---|---|
| Band 讀取 | ~166 s | **~39 s** | 4.2× |
| Inference（1849 patches × TTA）| ~8 min（batch=8） | **~2 min**（batch=64 + autocast） | |
| Post-process（NAS imread + imwrite）| ~78 s（阻塞） | **~26 s 可見**（其餘隱藏） | prefetch + async |
| **每場景總耗時** | **~9–10 min** | **~6 min** | CPU 並行競爭下實際值 |
| **66 場景/fold** | **~9.7 小時** | **~6 小時** | |
