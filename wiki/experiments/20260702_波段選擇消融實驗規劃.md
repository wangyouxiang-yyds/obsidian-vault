# 波段選擇消融實驗規劃：Sentinel-2 油汙指數應用可行性

## 目標

驗證 Zakzouk et al. (2024) 提出的 Sentinel-2 油汙光譜指數／精選波段組合，能否作為 DeepLabV3+ 訓練輸入，取代目前全 8 波段 baseline，並比較不同簡化策略的效果差異。

## 背景 / 來源論文

**Zakzouk et al. (2024)**, "Novel oil spill indices for sentinel-2 imagery: A case study of natural seepage in Qaruh Island, Kuwait", *MethodsX* 12, 102520.

論文用 Jeffries-Matusita (JM) 距離證明「精選少數波段」比「塞入全部波段」更能區分油/水，但論文本身只做到無監督分群驗證（K-means / GMM），沒有真正訓練語意分割模型。本實驗要驗證這個結論在監督式分割（DeepLabV3+）上是否依然成立。

## 波段對應（已依 wiki 現有資料確認，非猜測）

現有 8-band 資料集（MS6_sen2like，見 [[ms6_sen2like]]）波段順序：

| Index | 波段 | 波長(nm) |
|---|---|---|
| 1 | B01 | 443 |
| 2 | B02 | 492 |
| 3 | B03 | 560 |
| 4 | B04 | 665 |
| 5 | B08 | 833 |
| 6 | B8A | 865 |
| 7 | B11 | 1614 |
| 8 | B12 | 2202 |

論文使用的波段 B1/B2/B3/B8A/B11/B12 對應到現有資料集的 **index [1, 2, 3, 6, 7, 8]**（跳過 index 4 的 B04、index 5 的 B08）。

⚠️ **注意**：CLAUDE.md 裡記載的是「11 波段」規格（含紅邊 B05/06/07），但目前 server 實際訓練資料集（MS6_sen2like）是 **8 波段**，不含紅邊波段。本計畫以實際 8-band 資料集為準；CLAUDE.md 的波段表格已過時，之後有空應一併更新。

## 實驗設計（4 組對照）

| 組別 | 輸入內容 | Channel 數 | 對應現有 8-band index |
|---|---|---|---|
| **組 0 Baseline** | 現有全部 8 波段 | 8 | 1,2,3,4,5,6,7,8（沿用既有已完成訓練結果） |
| **組 1 Band Subset** | B1/B2/B3/B8A/B11/B12 原始反射率值（不做指數轉換） | 6 | 1,2,3,6,7,8 |
| **組 2 Index C** | R=(B1−B11)/(B1+B11)，G=(B1+B2)/(B3+B11)，B=(B1+B2)/(B3+B12) | 3 | 由 index 1,2,3,7,8 算出 |
| **組 3 Index D** | R=(B1−B11)/(B1+B11)，G=(B1+B2)/(B3+B11)，B=(B1+B2)/(B3+B8A) | 3 | 由 index 1,2,3,6,7 算出 |

## 實作方式（待 server 確認後選定）

**組 0**：直接沿用既有訓練結果作為比較基準，不需重跑——前提是先確定拿哪一組已完成的 baseline（目前 wiki 上並存多組：5-fold GB1.0、3-fold all-220 GB1.5、三策略系列等），需要到 server 上確認目前哪個 baseline 版本是最新、且跟後續三組實驗最能公平比較的。

**組 1**：兩種可能做法，需視 `deeplab_adapter.py` 現有架構決定：
- (a) 輕量做法：不重建 VRT/patch，只在 dataloader 讀取階段依 `band_indices` 做 channel 切片（新增一個 config 參數即可）
- (b) 重做法：另外產生 6-band 的 stacked TIF/VRT，走一次完整 pipeline（patch 座標理論上不用重跑，因為切割邊界跟波段數無關，只有最後 stacking 步驟要改）

**組 2/3**：需要新增一個指數計算步驟（可在 dataloader 讀取階段即時計算，也可預先算好存成新的 3-band TIF）。實作要注意：
- 分母可能為 0（nodata 像素）→ 需加 epsilon 避免除以零
- ResNet50 第一層 conv 需要從 8-ch 改為對應的 3-ch/6-ch（專案已有「8-ch 改造」的先例可直接參考，見 [[DeepLabV3+]]）

## 共同控制變因

- 同一批影像來源、同一個 train/val/test split（哪一組 split 待確認，見下方待確認事項）
- 同一組 HNM Golden Dataset（如果組 0 baseline 有用 HNM，組 1/2/3 也要用同一批，避免資料外洩或不公平比較）
- 同一模型架構（DeepLabV3+ ResNet50）、同一組超參數（class_weights=[13.0,1.0]、FocalLoss(gamma=2.0)、lr=5e-5 等完全比照 baseline，只有 input channel 數字必然不同）
- 同一評估指標：mIoU、Oil IoU、Precision/Recall/F1(Oil)

## 選用：低成本前置驗證（Optional Step 0）

在真正投入訓練前，可先比照論文方法，直接在現有資料集的 oil/background pixel 上算一次 JM 距離（4 組資料各算一次）。如果 JM 距離排序跟論文結論一致（Index C/D ≥ Band Subset ≥ Baseline），就有初步信心再去跑正式訓練；如果不一致，代表你的場景分布跟論文的 Qaruh Island 天然滲漏場景不同，要重新評估是否值得投入完整訓練資源。這一步成本很低（純統計計算，不需訓練模型），建議先做再決定要不要四組全上。

## 待確認事項（到 server 上確認）

1. 目前哪一組已完成的 baseline 訓練結果要拿來當「組 0」的比較基準（[[三策略比較實驗計畫]]／5-fold GB1.0／3-fold all220 等，目前有多組結果並存，需要挑一個公平對照組）
2. `deeplab_adapter.py` 的 dataloader 是否已支援 band_indices 這種 channel 篩選參數，或需要新增
3. 是否要重跑 patch coords / HNM，還是能直接沿用現有 patch 座標（理論上波段數變動不影響切割座標，只影響最後 tensor stacking，但需要在 code 裡實際確認）
4. Index 計算要放在 dataloader 即時算，還是預先產出新的 TIF（會影響 I/O 效能與磁碟空間佔用）

## 下一步

1. 到 server 上確認上述 4 項待確認事項
2. 先跑 Optional Step 0（JM 距離初步驗證，低成本）
3. 依驗證結果決定要不要四組全部進正式訓練，或優先跑最有希望的 1–2 組
4. 訓練完成後記錄每組結果，回填本頁「結果」章節，並更新 log.md
