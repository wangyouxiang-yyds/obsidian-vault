# JM 距離能否自動判別髒資料？可行性診斷（結論：不行）

## 1. 實驗背景

標注過程中，對主觀覺得「不好」的 scene，人工放了 `dirty.txt` / `blur.txt` 空旗標（blur 與 dirty 語意相近，本次分析合併視為單一「髒」標籤）。目前共 85 個髒 scene，其中 60 個有油。

[[20260702_波段選擇消融實驗規劃]] 中引用的 Zakzouk et al. (2024) 用 Jeffries–Matusita（JM）距離量化波段對「油 vs 海水」的可分性，作為選波段的依據。本次 side study 想仿照這個做法，量化判別哪些資料髒／乾淨，設想是：髒 scene 應該表現出某種可量測的光譜異常。

動手前先釐清一個關鍵前提：論文的 JM 量的是「油 vs 海水」的**可分性（separability）**，用來挑波段；而這裡想量的是「乾淨 vs 髒」的**異常度（abnormality）**。兩者的比較對象完全不同，這個落差正是後續失敗的根源，不是巧合。

實驗分支：B（[[GT_expand_pipeline]]，OIL_PROJECT_MutiBand_GT_expand），全程唯讀分析，不動訓練資料。

---

## 2. 方法架構：三個診斷框架

依序做了三層，逐步逼近「異常度」這個真正想量的東西：

```
框架 1：scene 級 oil-vs-bg JM
    → 直接套論文原始定義（油 vs 背景可分性），看能不能區分髒/乾淨 scene
    ↓ 結果：純亂猜，換方向
框架 2：patch 級三指標（oil-vs-bg JM / 銳利度 / 亮度）
    → 猜測「髒」可能同時反映在模糊度、過曝／雲亮這些視覺線索上
    ↓ 結果：略高於亂猜，但都不夠用
框架 3：anomaly-JM（patch 水體分布 vs 該感測器的乾淨參考水分布）
    → 換掉比較對象：不比「油 vs 水」，改比「這個 patch 的水 vs 正常的水」
    ↓ 結果：目前三者中最好，但仍然飽和、不足以自動判定
```

### 框架 1：scene 級 oil-vs-bg JM

直接沿用論文定義，對每個 scene 算 oil pixel 分布 vs background pixel 分布的 JM 距離，看這個「可分性」數值能不能拿來當「髒度」代理。

### 框架 2：patch 級三指標

因為訓練用的 fold 檔（358clean）已經先排除了髒 scene，沒有乾淨/髒可以對比，所以改成自己對全部 276 個有油 scene，用同一套 256 grid 規則重新生成含油 patch（不經過既有 fold 篩選），才能拿到 clean/dirty 都有的樣本。三個指標：oil-vs-bg JM、銳利度（variance of Laplacian，量模糊）、亮度（量過曝／雲的代理）。另外算了 bright_frac（高亮像素比例）當雲的代理指標。

### 框架 3：anomaly-JM

換掉比較對象：不再拿 oil vs background 比，而是把每個 patch 的「水體」像素分布，拿去跟**該感測器**（S2 / L8 / L9 各自建一個）的乾淨參考水分布比 JM 距離。邏輯是：如果一個 patch 的水看起來「不像正常的水」（被霧霾／薄雲污染、色偏），異常度應該偏高。同時逐波段拆解 clean 水 vs dirty 水的 JM，找大氣污染的光譜指紋。

## 4. 量化結果

### 框架 1：scene 級 oil-vs-bg JM

| 指標 | 值 |
|---|---|
| AUC（低 JM 預測髒） | **0.498** |
| JM 中位數（clean） | 1.751 |
| JM 中位數（dirty） | 1.756 |

AUC ≈ 0.50 = 純亂猜，兩組分布幾乎重疊。

### 框架 2：patch 級三指標

| 指標 | patch AUC | scene AUC | 說明 |
|---|---|---|---|
| oil-vs-bg JM（低→髒） | 0.579 | 0.559 | 略高於亂猜 |
| 銳利度（低→髒，variance of Laplacian） | 0.588 | 0.569 | 略高於亂猜 |
| 亮度（暗→髒） | **0.645** | 0.593 | 三者最高，但方向和預期相反——髒 patch 偏**暗**不是偏亮 |
| bright_frac（雲代理，高→髒） | 0.486 | 0.538 | 幾乎無效，雲代理指標抓不到 |

雲代理 `bright_frac`（高亮像素比例）無效，說明「髒」的視覺成因不是單純的雲亮異常；亮度指標最好，但方向反直覺（髒偏暗），暗示「髒」裡混了不只一種成因（見第 5 節）。

### 框架 3：anomaly-JM

| 指標 | 值 |
|---|---|
| patch AUC | **0.642** |
| scene AUC | 0.603 |
| anomaly-JM 中位數（clean） | 1.953 |
| anomaly-JM 中位數（dirty） | 1.985 |

三者中最好，但飽和：clean 和 dirty 中位數都貼近 JM 距離上限 2.0（1.953 vs 1.985），代表大多數 patch（不論髒不髒）水體本身自然變異就已經很大，把上限撐滿了，區分空間被壓縮。

**逐波段 JM（clean 水 vs dirty 水），大氣霧霾的光譜指紋：**

| 波段 | JM |
|---|---|
| B01（氣溶膠） | 0.136 |
| B02 | 0.107 |
| B03 | 0.114 |
| B04 | 0.125 |
| **B08（NIR）** | **0.144** |
| B8A | 0.122 |
| B11（SWIR1） | 0.071 |
| B12（SWIR2） | 0.078 |

B01 + B08 偏移最大、SWIR（B11/B12）最小——這正是大氣霧霾/薄雲的典型光譜特徵（短波紅外穿透力強，受氣狀污染物影響最小），說明 anomaly-JM 抓到的主要是**大氣層面**的異常，而非「肉眼判髒」的全部成因。

## 5. 觀察與結論：為什麼 JM 距離不行

使用者實際的「髒」標記準則（本人說明）：肉眼看油汙跟背景太像、藏在雲朵下不明顯、跟旁邊區域太相似。這是**視覺 + 空間 + 多因素**的整體判斷，而 JM 只看每個像素的**數值分布**，看不到形狀、紋理、上下文關係。

更關鍵的是，兩種常見的「髒」成因，在 JM 上的效果方向相反、會互相抵消：

- **太相似的 faint 油**（油跟背景光譜太接近）→ oil-vs-bg JM **偏低**
- **藏在雲下的油**（實際上框到的是雲，不是油）→ 雲跟水的光譜差異很大 → JM **反而偏高**

兩群樣本互相抵消，導致 oil-vs-bg JM 的區分力卡在 AUC ≈ 0.58 上不去。

進一步檢查框架 2（低 oil-JM）與框架 3（高 anomaly-JM）抓到的樣本，發現兩者互補（各自抓到 faint 油 / 雲霧兩種不同成因），但前 100 名重疊只有 9 個。嘗試線性組合兩個分數，AUC 反而降到 ~0.53，判斷這是建模假象（過擬合到訓練分布），不是真的互補增益。

**結論一句話：這是「標籤性質」與「工具性質」的錯配，不是 JM 距離本身壞掉。** 使用者的髒標籤是視覺 + 空間的整體判斷，任何單一的逐像素光譜統計量，都無法乾淨復現這種判斷。

---

## 6. 副產物

- **弱分流工具**：anomaly-JM 分數最高的一批 patch，髒樣本富集約 2.8 倍（前段高分樣本中約 40% 是髒，對照基準髒佔比僅 14%）。可以作為人工複查排序的優先順序參考，但**不能**用來自動篩除資料。
- **副確認**：不論髒不髒，油 patch 的 oil-vs-bg 可分性中位數都 > 1.1（滿分 2），說明目前的油汙標註在光譜上是「實的」，不是雜訊——這個 side study 沒有動搖既有標註品質的信心，只是確認了「髒/乾淨」這個標籤本身無法用 JM 自動化。

---

## 7. 後續改進方向

- [ ] 若要自動化「肉眼髒」判斷，需換工具類別：學習型視覺模型（小 CNN 直接吃 patch 影像 + 髒標籤，可學雲的外觀、油膜形狀、faint 油的視覺線索），而非手算光譜統計量
- [ ] 或改用專用雲遮罩工具（Fmask / s2cloudless），先把「藏在雲下」這一類成因單獨處理掉
- [ ] anomaly-JM 高分 patch 清單可保留作人工複查排序輔助（見第 6 節），但不建議再投入時間優化 JM-based 自動判髒——本次診斷判定手算光譜公式這條路線到此收尾

## 8. 附帶發現：SSL backbone band_map 錯位（bug）

驗證框架 3 的波段身分時，順手發現 SSL4EO backbone 的 `band_map` 設定有誤：

- 實際 VRT 8 通道為 `[B01, B02, B03, B04, B08, B8A, B11, B12]`（不含 B05，含 B8A）
- 但 ssl4eo yaml 的 `band_map=[0,1,2,3,4,7,11,12]` 是假設輸入含 B05 的 12 通道規格下反推的索引，套在本專案實際的 8 通道排序上會錯位：通道 5（本專案的 B08）被當成 SSL 的 B5（紅邊）濾波器輸入，波段身分對不上（通道 6 的 B8A→SSL B8 誤差可忽略）
- 正確應為 `band_map=[0,1,2,3,7,8,11,12]`

影響範圍：目前正在跑的 arm A（norm10k）/ arm B（frozen）SSL4EO 相關實驗。已記錄在正本 `oil_spill_project_status.md` ⑤-6，待決策，此處僅在 vault 留一筆索引，細節與後續處置以正本為準。相關背景見 [[deeplabv3plus_358clean_overfitting_改善方向]] 中 SSL4EO-S12 預訓路線的討論。
