# iForest 全場景改進：乾淨水體訓練 + Percentile 聚合

## 問題根源分析

### 為什麼 GT 輔助判得好，全場景卻判不到油汙？

**GT 輔助情境（效果好）**
```
訓練資料 = GT 框框旁邊的乾淨海水
→ iForest 學到「這片海域的正常海水長這樣」
→ 油汙光譜偏離 → 被標為異常 ✓
```

**全場景情境（效果差）**
```
訓練資料 = 隨機 500 個 patch（可能含油汙）
→ iForest 學到模糊的「場景平均分布」
→ 油汙佔全場景 0.01~0.1%，根本不是最異常的東西
→ 陸地邊緣、雲影、淺水更異常 → 油汙被淹沒 ✗
```

### 三個根本原因

| 原因 | 說明 |
|------|------|
| 污染比例 | 全場景油汙 < 0.1%，iForest 的 contamination='auto'（≈10%）設定不匹配 |
| 競爭異常 | 陸地邊緣、雲影、淺水比油汙更「統計異常」 |
| 分數被均值抹平 | Patch 內油汙佔 5%，200 個採樣像素中約 10 個是油，mean score ≈ 水的分數 |

---

## 用戶想法：切小塊 → 各自判斷 → 拼回大圖

核心概念正確，現有腳本已在做這件事：
```
全場景 → 滑動窗口 256×256（stride=192）→ 每個 patch 評分 → 拼回 score_map
```

真正的問題在於 **iForest 訓練資料的品質**，而非是否切小塊。

GT 輔助之所以有效，是因為訓練資料是「確定乾淨的海水」，而非因為切了小塊。

---

## 改進方案

### 改進一：只用乾淨海水 Patch 訓練

**Pass 1 篩選條件（三重）**

```python
# ① NDWI > 0.1：確定是深水（排除陸地、沿岸淺水）
deep_water = (ndwi > NDWI_CLEAN_THRESH) & ~cloud

# ② 無雲：B11 > 0.05 AND B02 > 0.20
cloud = (b11 > 0.05) & (b02 > 0.20)

# ③ patch 內光譜均勻（std < 0.05）：排除油汙、船隻、渦流
per_band_std = px_valid.std(axis=0).mean()
is_uniform = per_band_std < UNIFORM_STD_THRESH
```

需滿足 70% 像素通過 ① + ② 才使用。

**效果**

| 地物 | 是否被用來訓練 |
|------|--------------|
| 深海乾淨海水 | ✓ 被選入 |
| 油汙 | ✗ std 高 |
| 陸地 | ✗ NDWI < 0.1 |
| 雲 | ✗ 雲遮罩 |
| 沿岸淺水 | ✗ NDWI < 0.1 |
| 船隻/渦流 | ✗ std 高 |

### 改進二：5th Percentile 取代均值

```python
# 舊版：mean（油汙被海水稀釋）
mean_s = float(scores[offset:offset + n_px].mean())

# 新版：5th percentile（保留最異常像素的信號）
agg_s = float(np.percentile(patch_scores, 5))
```

**原理**
- Patch 內油汙 5%，採樣 200 像素 → ~10 個是油汙
- mean：被 190 個水像素拉平
- 5th percentile：正好落在那 10 個油汙像素的分數區間

## 實作細節

### 參數設定

```python
NDWI_CLEAN_THRESH  = 0.1    # 深水門檻（0.2 太嚴，全場景 0 個 patch 通過）
UNIFORM_STD_THRESH = 0.05   # 光譜均勻度（0.02 太嚴）
CLEAN_WATER_RATIO  = 0.7    # patch 內 70% 需通過乾淨水體條件
SCORE_AGG_PERCENTILE = 5    # 取最異常前 5% 的像素分數
```

### Fallback 機制

若乾淨水體 patch < 50 個（嚴重雲覆蓋或全場景陸地），自動退回原始 NDWI>0 模式：

```python
if len(all_pixels) < 50:
    print('⚠ 乾淨水體 patch 不足，退回一般水體模式')
    # 使用 NDWI > 0 重新收集
```

### 遇到的問題

初始設定 NDWI_CLEAN_THRESH=0.2 + std<0.02 + 80% 比例，導致 NOAA 墨西哥灣場景全部 3249 個 patch 被篩掉（0 個通過），訓練失敗。調整為上表參數後修正。
