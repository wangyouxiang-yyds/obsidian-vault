# 分割任務中處理類別不平衡的損失函數設計

## 一句話定義

在正樣本（油汙）遠少於負樣本（背景海面）的語意分割任務中，標準 cross-entropy 會讓模型偏向「全猜背景」也能拿到高分，因此需要專門設計的損失函數或類別權重機制來讓模型「更在意漏抓正樣本」。

## 詳細說明

### 三種常見機制，處理層級不同

| 機制 | 處理層級 | 核心手法 | 代表方法 |
|---|---|---|---|
| Class re-weighting | 外部、資料統計 | 依類別出現頻率反比加權 loss | `class_weights=[13.0, 1.0]` |
| Focal 調變 | 損失函數、per-pixel | 用 `(1-p)^gamma` 壓低易分類樣本的梯度貢獻，聚焦難分類像素 | FocalLoss(gamma=2.0) |
| 相似度損失的不對稱化 | 損失函數、FP/FN 加權 | 直接在 overlap-based loss（Dice）中引入 α/β 分別加權 FP、FN | Tversky loss |

三者可以疊加使用（本專案的 OSDMamba 就是 Dice + FocalLoss 疊加），但也可能疊床架屋：如果 class_weight 已經把 Oil 類別放大 13 倍，FocalLoss 的 gamma 又再做一次「聚焦難樣本」，兩者的效果邊界會變得難以拆解與解釋。

### Dice Loss vs Tversky Loss

Dice 係數（= F1 score）對 FP 與 FN 給予相同權重：

```
D(P, G) = 2|P∩G| / (|P| + |G|)
```

Tversky index（[[Salehi2017_TverskyLoss]]）引入 α、β 兩個超參數獨立控制 FP、FN 的懲罰：

```
S(P, G; α, β) = |P∩G| / (|P∩G| + α|P\G| + β|G\P|)
```

- α = β = 0.5 時，Tversky index 退化為 Dice 係數
- α 越小（β 越大）→ 越加重 FN 懲罰 → 模型訓練後偏向高召回率
- 這個超參數的意義**直接對應到 precision/recall trade-off**，比 FocalLoss 的 gamma（控制的是「難易樣本」而非「FP/FN 不對稱」）更貼近不平衡分割任務的核心訴求

### 為何「全域固定的 α/β 或 class_weight」有其局限

無論是 class_weight 還是 Tversky 的 α/β，通常都是對整個資料集設一個全域固定值。但不平衡的嚴重程度可能因場景而異（例如大片浮油場景 vs. 零星油膜場景），全域固定的懲罰權重不一定是每張影像的最適值。這是這類方法共同的隱含假設，也是後續研究（如 Focal Tversky Loss，引入 per-pixel 的 γ 項）試圖解決的方向。

## 在本專案的應用

- [[DeepLabV3+.md]] / [[OSDMamba.md]]：models/ 頁面記錄的仍是較早期設計——`class_weights=[13.0,1.0]`+FocalLoss（DeepLabV3+）、Dice(對稱 α=β=0.5)+FocalLoss（OSDMamba）
- [[GT_expand_pipeline.md]]（第三-c 節）：**目前現行研究線已經實測過三種機制的橫向比較**——baseline（無加權，pooled oil IoU=0.332）、cw31（class_weights=[3.0,1.0]，均值 0.394 但 Wilcoxon 不顯著、tiny 油汙場景顯著變差）、tversky（Tversky loss α=0.3/β=0.7，pooled 0.3915，**唯一通過雙 gate**）。目前判定 tversky 優於 cw31，已成為標竿 loss 設定並帶入下一輪架構消融（`tversky_v3plus`）
- [[hard_negative_mining.md]]：從**資料層級**處理不平衡（挖掘難分類的背景樣本納入訓練），與損失函數層級的機制是互補關係，不是替代關係

**這組實測結果本身就是一個值得記住的發現**：全域 class_weight（cw31）雖然平均分數不差，但代價是犧牲了 tiny 油汙場景的表現（trade-off 而非全面提升）；而 Tversky 的 FP/FN 不對稱化在同一份資料上通過了更嚴格的統計判準。這與 [[Salehi2017_TverskyLoss]] 論文本身「全域固定 α/β 也有其局限」的假設殺手觀察一致方向——但本專案的 tversky 目前仍是全域固定 α，尚未驗證是否還有進一步空間（例如按場景油汙密度動態調整）。
