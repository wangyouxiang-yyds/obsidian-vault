## 2. VAE 是什麼？

### 從 Autoencoder 說起

Autoencoder（AE）是一個「壓縮再還原」的神經網路：

```
輸入 x (224維)
    ↓ Encoder
潛在向量 z (10維)   ← 資訊被壓縮
    ↓ Decoder
重建 x̂ (224維)
```

訓練目標：讓 x̂ 盡量接近 x（最小化重建誤差 MSE）。

Encoder 被迫學「什麼特徵最重要」，學到的 z 就是一個緊湊的語意表示。

### VAE 的「V」加在哪

VAE = Variational Autoencoder。它在 AE 的基礎上加了一個約束：

| 面向 | 普通 AE | VAE |
|------|---------|-----|
| z 是什麼？ | 一個固定向量 | **一個分布** N(μ, σ²) |
| Encoder 輸出 | z | **μ 和 σ**（分布的兩個參數）|
| z 怎麼取得？ | 直接 | 從 N(μ, σ²) **抽樣** |
| Loss | 只有 MSE | MSE + **KL 散度懲罰** |

KL 散度的作用：強迫每個像素的潛在分布 N(μ, σ²) 趨近標準高斯 N(0, I)。這個約束讓整個 latent space 形成一個有結構的高斯分布。

### Reparameterization Trick

直接「從 N(μ, σ²) 抽樣」無法做反向傳播（抽樣操作沒有梯度）。VAE 的巧思：

```
z = μ + σ ⊙ ε，其中 ε ~ N(0, I)
```

- 隨機性由 ε 承擔
- 梯度只流過 μ 和 σ（可微）
- 數學等價，但可以訓練

### Loss = λ₁·MSE + λ₂·KL

```
L = λ₁ · ‖x - x̂‖²   (重建誤差)
  + λ₂ · KL[N(μ,σ²) ‖ N(0,I)]  (讓 latent 趨近高斯)
```

KL 散度的展開式（對角協方差 σ² 情形）：

```
KL = ½ Σ (σ² + μ² - 1 - log σ²)
```

這是解析解，不需要蒙特卡羅估計。

## 3. 為什麼要 VAE + RX 兩個串起來？

### 單獨用 RX 的問題

RX（Reed-Xiaoli）是高光譜偵測的經典方法，核心公式：

```
r(x) = (x - μ̂)ᵀ Σ̂⁻¹ (x - μ̂)   ← Mahalanobis 距離
```

**問題 A**：RX 假設背景服從高斯分布。高光譜資料（224 波段）實際上是多峰分布（水、植被、土壤、陰影各自成群），不是高斯。

**問題 B**：高維空間的協方差矩陣 Σ̂ 估計困難。224×224 的矩陣需要 >>224 個樣本才能可靠估計，而且很容易接近奇異（singular），求逆不穩定。

### 單獨用 VAE 的問題

只用重建誤差（‖x - x̂‖²）當異常分數也行，但：

- 浪費了 VAE 已經學好的 latent 高斯結構
- 重建誤差受 λ₁/λ₂ 權衡影響，不夠穩定

### 合起來的邏輯

```
VAE Encoder → latent z ~ N(μ, σ²)
                        ↑
          KL 約束強迫這裡是高斯
                        ↓
      在 latent space 做 RX → RX 的高斯假設在這裡成立
      維度只有 10 → Σ 估計穩定
```

Deep-RX 的核心思想：**用 VAE 把「對 RX 不友善的分布」轉換成「對 RX 友善的分布」**，再讓 RX 在轉換後的空間裡發揮。

## 4. Deep-RX 對 San Diego I 的完整流程

### 資料集規格

San Diego I：100×100 像素，189 有效波段（AVIRIS 高光譜），異常目標為飛機（3 架，各自形狀不同）。

### 訓練階段：VAE 學習背景光譜結構

```
輸入影像 X shape = (100, 100, 189)
    ↓ 攤平
(10000, 189)   ← 每列是一個像素

VAE 架構（全連接 MLP）：
    Encoder: 189 → 128 → 64 → 32 → [μ(10), σ(10)]
    Decoder: 10 → 32 → 64 → 128 → 189

訓練：
    optimizer = SGD（mini-batch）
    epochs = 300
    所有 10,000 個像素都會在每個 epoch 被看到
```

每個 epoch，VAE 對「包含飛機」的全部像素都學一遍。但飛機只佔 10,000 個像素中的一小部分（<<1%），VAE 的 loss 主要由大量背景像素決定，它學到的是**背景的光譜結構**。飛機像素的存在對 loss 影響微小，VAE 基本上忽視它們。

### 推論階段：丟掉 Decoder，只用 Encoder

```
訓練好的 VAE → 只保留 Encoder 的 μ 分支（丟棄 σ 分支和整個 Decoder）

全部 10,000 個像素 → Encoder → μ 向量
    每個像素 xᵢ → μᵢ ∈ ℝ¹⁰

→ 得到 latent feature matrix Z shape = (10000, 10)
```

### 計算 RX 異常分數

```
在 latent space 估計背景統計量：
    μ̂_b = mean(Z, axis=0)    ← 10 維均值向量
    Σ̂_b = cov(Z)             ← 10×10 協方差矩陣

對每個像素計算 Mahalanobis 距離：
    r(xᵢ) = (μᵢ - μ̂_b)ᵀ Σ̂_b⁻¹ (μᵢ - μ̂_b)

→ anomaly score map shape = (100, 100)
→ AUC = 0.991（論文 Table 1）
```

## 5. 訓練 / 測試同一張的爭議（誠實版）

### 論文裡實際寫了什麼

- **§3.1**：用 X 表示輸入高光譜影像
- **§3.2**：用 T（to be detected）表示待偵測影像
- **Algorithm 1**：只有一個輸入 X，沒有分 train set / test set
- **§4.1**：三個資料集（San Diego I/II, HyperUrban）各自獨立介紹
- **§4.2**：「The network was trained for 300 epochs」——「The network」用單數，暗示每個資料集各自訓一個 VAE

### 論文沒明說的

- X 和 T 是否相同（= 是否同張影像訓+測）
- 是否每個 dataset 各自獨立訓練一個 VAE
- 訓練時是否包含全部像素（包含目標飛機）

### 合理推論

**高機率是：每張影像各自訓一個 VAE + 同張影像推論（per-scene transductive）**

推論依據：
1. Algorithm 1 只有一個輸入，暗示 X = T
2. 三個 dataset 波段數不同（189 / xxx / xxx），VAE 輸入維度不同，無法共用一個 VAE
3. 高光譜異常偵測（HAD）領域的慣例就是 per-scene：每張影像單獨訓練模型然後對同張影像偵測

## 6. 「這算不算作弊？」的誠實討論

### 嚴格定義：不算作弊

VAE 在訓練時從未看過 Ground Truth label（哪個像素是飛機）。它不知道「飛機」這個概念。它只是學「這 10,000 個光譜向量的整體分布長什麼樣」。

這類似於 [[iforest_架構與運作機制]] 第 5 節討論的 iForest transductive analysis——學的不是「x → y 對應」，而是「資料的幾何結構」。

### 但評估方式有實質問題

**問題 A：AUC 0.991 的泛化意義有限**

「AUC = 0.991」只證明：在 San Diego I 這張影像上，VAE 學到的背景結構能讓飛機的 Mahalanobis 距離排在水泥地 / 跑道 / 草地之前。這不等於：對一張新的機場影像，同樣的架構也能做到 0.991。

**問題 B：沒做 cross-scene 評估**

論文沒有做 Setup B 形式的評估：「在影像 A 訓練 VAE，在影像 B 偵測」。這種評估才能驗證「學到的東西是否可遷移」。

**問題 C：基準線比較不公平**

論文拿 Deep-RX（每張影像都訓了 300 epochs）去比 GRX/LRX/KRX（傳統方法，沒有機會「學」這張影像）。傳統方法在「不學習」的前提下輸了，這並不驚人——更有意義的比較是 Deep-RX vs 其他「也對同一張影像做 per-scene fitting」的方法。

### HAD 領域的慣例與侷限

Per-scene evaluation 是 HAD 領域的通行做法，不只 Deep-RX 這樣——大部分論文都這樣，所以論文被接受不代表這個評估方式本身沒有缺陷。

學術界確實有人批評：per-scene training 讓方法看起來比實際上更強，而且讓研究結果難以在真正的新場景部署時復現。

### 對本專案的直接影響

論文 AUC = 0.991 是在 AVIRIS 機載高光譜（189 波段，100×100）、已知只有飛機異常的乾淨場景下拿到的。本專案的 v9 面對的是 Sentinel-2 多光譜（8-11 波段，真實海洋場景，雲/陸地/船都混在裡面）。論文的 0.991 不能直接套用到 v9 的期望值上。

## 7. 使用者的核心洞察：Background-Only Training

### 使用者的提問

> 如果只訓乾淨水體再對全圖打分，可不可以？

### 這個策略的學術名稱

**Background-Only Training** / Clean Background Bootstrap：

訓練 VAE 時，只餵入「確定是背景」的像素，讓 VAE 學到一個純淨的背景分布。推論時，對全部像素（包含潛在油汙）計算 Mahalanobis 距離。

### 雞生蛋問題的解

「你說只給乾淨水體，可是你不知道哪裡是油汙、哪裡才算乾淨水體」——這是合理的疑慮。

但關鍵是：**不需要知道「哪裡 IS 油汙」，只需要知道「哪裡絕對不是油汙」**。

可以用物理條件排除：

| 條件 | 說明 |
|------|------|
| NDWI 高（> 0.3） | 確定是水體，不是陸地 |
| B02（藍）低（< 0.1） | 深水，非淺灘反射 |
| B11（SWIR1）極低（< 0.05） | 乾淨水，油膜會讓 SWIR 升高 |
| 空間紋理均勻（local std 低） | 避開雲邊、浪花、船跡 |

這些條件的交集——即便影像上有油汙——仍然能找出「幾乎確定是乾淨水體」的像素，作為 VAE 的訓練樣本。

### v9 已經實作這個洞察

v9 pipeline 的 `patch_is_scoreable` 機制 + `loose_mask` 就是 Background-Only Training 的工程實作：

| 面向 | 論文 Deep-RX | v9 DeepRX pipeline |
|------|-------------|---------------------|
| VAE 訓練資料 | 全部像素（包含飛機目標） | 通過 `patch_is_scoreable` 的**乾淨水體 patch** |
| 對 contamination 的抵抗力 | 弱（若目標佔比高則失效） | 強（目標被排除在訓練之外）|
| 混雜場景適應性 | 低（純高光譜機載，場景乾淨）| 高（真實衛星影像含雲/陸地/船）|
| 訓練像素的品質控制 | 無 | NDWI + B02 + B11 + 紋理多條件篩選 |

**這是 v9 在概念上比論文 Deep-RX 更進一步的地方**——不是因為模型更複雜，而是訓練資料的篩選策略更嚴謹。

### Background-Only Training 的風險

| 風險 | 說明 | 對應措施 |
|------|------|---------|
| 篩太鬆 | 油汙污染訓練集 → VAE 學到的「正常」包含油汙 → 偵測失效 | 多條件交集篩選（NDWI + B02 + B11 + std）|
| 篩太嚴 | 訓練像素不足 → VAE 學不好 → latent space 結構差 | 降低門檻，但需 sensitivity 分析 |
| 乾淨水體 VAE 對非水體打高分 | VAE 學到的是「乾淨水體的分布」，對陸地/雲/油汙都會輸出高 Mahalanobis 距離 | 需要 NDOI z-score 等二次篩選 |

### 為什麼 v9 還需要 NDOI z-score 後過濾

Background-Only Training 的 VAE 會對「所有非乾淨水體」都給出高異常分數——包括陸地、雲邊、船隻、以及油汙。這就是「高召回、低精確」的問題。

v9 用 NDOI（Normalized Difference Oil Index）z-score 做二次驗證：即便 Mahalanobis 分數高，若該像素的光譜特徵不符合油汙的物理特性（SWIR 吸收、藍光抑制），就不列入最終偵測結果。

**邏輯**：VAE+RX 負責找「不像乾淨水體的」（高召回），NDOI z-score 負責確認「是不是油汙」（提高精確度）。

## 8. 論文 vs 真實場景的複雜度差異（為什麼論文 AUC=0.99 但 v9 Recall 只有 0.37）

> 本節源自 2026-06-17~2026-06-18 Q&A，釐清「論文 0.99 怎麼來的」與「為什麼我的結果只有 0.37」之間的本質落差。

### 核心論點

論文 San Diego II 是「one normal type, one anomaly type」的玩具場景；真實 S2 海洋場景是「multi-anomaly type, only want one」的困難場景。

### 場景內容對比

| 面向 | 論文 San Diego II | v9 NOAA Gulf of Mexico |
|------|-------------------|------------------------|
| 影像大小 | 100 × 100 像素（1 萬像素）| ~12,000 × 10,000 像素（1.2 億像素）|
| 光譜維度 | 189 波段（AVIRIS 高光譜）| 8 波段（Sentinel-2 多光譜）|
| 「正常」種類 | 1 種（機場水泥地 + 跑道）| 多種（深海水、淺水、碎浪、海霧）|
| 「異常」種類 | 1 種（飛機）| 6 種（油汙、厚雲、薄雲、陸地、船隻、太陽耀斑）|
| 目標異常佔比 | < 1%（飛機）| < 0.1%（油汙，在所有「異常」中佔少數）|
| 場景競爭者 | 無（只有飛機這一種異常）| 5 種非目標異常（都比油汙更常見）|

### RX 分數分布對比

```
論文 San Diego II：
  背景分數  │████████████████████░░░░░░░░░░
  飛機分數  │░░░░░░░░░░░░░░░░░░░░████████████
            └─────────────────────────────→ 分數
              ↑                  ↑
            兩群清楚分離，門檻容易設
  任意門檻都能同時：高 Recall 飛機 + 低 FP 背景

真實 S2 場景：
  乾淨水體  │████████
  深水/碎浪 │     ███████
  薄雲      │          █████████
  太陽耀斑  │               ████████████
  船隻      │                   █████
  油汙      │                       ███████
  陸地      │                            █████████
            └──────────────────────────────────→ 分數
              ↑
            連續漸層，沒有明顯斷點
            設高門檻：錯過油汙 → Recall 低
            設低門檻：抓到太多非油異常 → Precision 低
```

### 評估指標差異

| 指標 | 論文 AUC | v9 Recall/Precision |
|------|----------|---------------------|
| 衡量什麼？ | **排名能力**（飛機分數整體 > 背景分數）| **實際偵測能力**（top-K 有多少真的是油汙）|
| 幾乎對任何分離場景都能 0.99 | ✅ 兩群分離就夠 | ❌ 需要油汙分數 > 所有其他異常 |
| 競爭異常影響 | 幾乎沒有（沒有競爭異常）| 嚴重（5 種競爭異常把油汙「淹沒」）|
| 對「門檻選擇」敏感嗎？ | 不敏感（AUC 是面積，不依賴單一門檻）| 非常敏感（同一分數分布下，門檻 ±0.1 可讓 Recall 差 2×）|

### 光譜維度差異的影響

| 面向 | 189 維（高光譜）| 8 維（S2 多光譜）|
|------|----------------|-----------------|
| 特徵可分性 | 每種物質有獨特的「光譜指紋」| 波段太少，油汙與薄雲光譜差異不顯著 |
| Mahalanobis 距離的效果 | 能精確定位到高維空間裡的「罕見方向」| 8×8 協方差矩陣，維度限制了區辨力 |
| KL 約束效果 | 高維 latent 能捕捉精細結構 | 低維 latent 壓縮過多，細節丟失 |

### 總結表格

| 比較項目 | 論文 San Diego II | v9 S2 場景 |
|----------|-------------------|------------|
| 影像大小 | 1 萬像素 | 1.2 億像素 |
| 光譜維度 | 189 維 | 8 維 |
| 「正常」種類 | 1 種 | 多種 |
| 「異常」種類 | 1 種（飛機）| 6 種（只想要油汙 1 種）|
| 分數分布 | 兩群清楚分離 | 連續漸層 |
| 評估指標 | AUC（排名）| Recall/Precision（top-K 正確率）|
| 競爭者 | 無 | 5 種非目標異常 |
| 最終結果 | AUC = 0.991 | Recall = 0.371 |

### 結論

論文 AUC=0.99 = 在一個「幾乎不可能失敗」的簡單場景裡取得勝利。v9 Recall=0.37 = 在一個真實複雜場景裡的誠實表現。這不是 VAE+RX 架構不夠強，而是這個架構**天生不適合 multi-anomaly 場景**——它只能找「不像背景的東西」，無法區分「我想要的異常」vs「我不想要的異常」。

## 9. v9 在「同影像內」的完整 12 步驟流程

> 本節源自 2026-06-17~2026-06-18 Q&A，釐清 v9 pipeline 的實際運作邏輯。

### 釐清常見誤解

| 誤解 | 正確理解 |
|------|---------|
| ❌ 每個 patch 各自訓一個 VAE | ✅ 一張影像只訓一個 VAE |
| ❌ 所有影像共用一個 VAE | ✅ 每張影像獨立訓一個 VAE |
| ✅ VAE 訓練資料 = 該影像通過篩選的 patches 的水體像素池 | 對 |

### 完整 12 步驟

```
影像 X（12,000 × 10,000 × 8 波段）
         │
Step 1   ↓  Sliding window 切 patches（256×256，stride 128）
         │  → 約 3,000 個 patch
         │
Step 2   ↓  patch_is_scoreable 篩選
         │  → 保留「水佔比 > 閾值，NDWI 夠高，無濃雲」的 patch
         │  → 約 1,500 個乾淨水體 patch
         │
Step 3   ↓  從 1,500 個 patch 取水體像素 → 池化
         │  → 約 500 萬個「確定是乾淨水體」的 8 維像素向量
         │
Step 4   ↓  訓練 VAE（50 epochs，mini-batch SGD）
         │  → VAE 學會這張影像的「正常水體光譜分布」
         │
Step 5   ↓  VAE 學會：8 維光譜 ─→ 10 維 latent N(μ, σ²)
         │  latent 空間因 KL 約束，趨近高斯分布
         │
Step 6   ↓  拿 Encoder 對全部 3,000 patch 的所有像素編碼
         │  每個像素 xᵢ → μᵢ ∈ ℝ¹⁰
         │  → 全圖 latent 矩陣 Z
         │
Step 7   ↓  估計背景統計量（只用通過篩選的 1,500 patch）
         │  μ̂_b = mean(Z[水體像素])
         │  Σ̂_b⁻¹ = inv(cov(Z[水體像素]))（10×10 矩陣）
         │
Step 8   ↓  全部像素算 Mahalanobis 距離
         │  r(xᵢ) = (μᵢ - μ̂_b)ᵀ Σ̂_b⁻¹ (μᵢ - μ̂_b)
         │  → 每個像素得到異常分數
         │
Step 9   ↓  Patch 級聚合：5th percentile
         │  每個 patch 有 65,536 個像素分數
         │  取最異常的 5%（3,277 個）的最小值作為 patch 分數
         │  → 約 3,000 個 patch 分數
         │
Step 10  ↓  EM 自動門檻：2 高斯混合模型
         │  把 3,000 個 patch 分數擬合成 2 個高斯
         │  → 找兩個高斯的交點作為分類門檻
         │  → 高於門檻 = 異常 patch 候選
         │
Step 11  ↓  NDOI z-score 後過濾（油汙物理驗證）
         │  對異常 patch 候選計算 NDOI（Normalized Difference Oil Index）
         │  z-score ≥ 0.5 才保留
         │  → 排除雲邊緣、海岸線、太陽耀斑
         │
Step 12  ↓  輸出結果
            .txt：異常 patch 座標列表
            .tif：全圖 score map
            .jpg：偵測結果疊加對比圖
```

### 4 大階段濃縮

```
┌─────────────────────────────────────────────────────────────┐
│ 階段 1：準備乾淨訓練資料                                     │
│   Step 1~3：Sliding window → 篩選乾淨水體 → 池化像素        │
├─────────────────────────────────────────────────────────────┤
│ 階段 2：學習正常分布                                         │
│   Step 4~5：訓練 VAE → 得到 latent 高斯結構                 │
├─────────────────────────────────────────────────────────────┤
│ 階段 3：找異常                                               │
│   Step 6~10：全圖編碼 → 估統計量 → Mahalanobis → Patch 聚合 → EM 門檻│
├─────────────────────────────────────────────────────────────┤
│ 階段 4：油汙物理驗證                                         │
│   Step 11~12：NDOI z-score 過濾 → 輸出結果                  │
└─────────────────────────────────────────────────────────────┘
```

### 元件分工表格

| 元件 | 負責什麼 | 輸入 | 輸出 |
|------|---------|------|------|
| Sliding Window + patch_is_scoreable | 把影像切成 patch，篩出「確定是乾淨水體」的訓練樣本 | 全圖像素 | ~1,500 個乾淨 patch |
| VAE | 學習正常水體的光譜分布，把 8 維壓成 10 維高斯 latent | 500 萬水體像素 | 訓練好的 Encoder |
| RX / Mahalanobis | 在 latent 空間量「這個像素和正常水體差多遠」| 全圖 latent Z | 每像素異常分數 |
| EM（2 高斯混合）| 自動找 patch 分數分布的分割點，不需人工設門檻 | 3,000 patch 分數 | 分類門檻 |
| NDOI z-score | 驗證異常 patch 是否符合油汙的物理光譜特性 | 異常 patch 候選 | 最終偵測結果 |

## 10. 為什麼跨影像不共用 VAE

> 本節源自 2026-06-18 Q&A，說明 per-scene 訓練的必要性。

### 不同影像水體分布差異的原因

| 來源 | 影響 | 後果 |
|------|------|------|
| 大氣狀況（霾、霧、氣膠量）| 影響各波段的大氣散射量，藍光影響最大 | 相同水體，B02 數值在不同影像可差 30% |
| 太陽高度角（季節 / 時刻）| 改變入射光角度，影響鏡面反射率 | 正午 vs 清晨，B11 SWIR 值可差 2× |
| 海面狀況（海況、浪高、碎浪）| 浪花增加白光反射，改變各波段比例 | 同一海域，不同風速下 B02 差 15~20% |
| 感測器差異（S2A vs S2B）| 兩顆衛星的 SRF 有細微差異 | 同一天同一區，A/B 星各波段可差 2~5% |

### 跨影像共用 VAE 的後果

```
情境：用影像 A（平靜、低霾）訓的 VAE → 套用到影像 B（波浪多、高霾）

影像 A 水體分布：μ_A = [0.04, 0.03, 0.02, ...]   ← 低值，乾淨水
影像 B 水體分布：μ_B = [0.08, 0.07, 0.06, ...]   ← 值偏高，因大氣影響

VAE 學到的「正常」= μ_A
影像 B 所有水體像素 → Mahalanobis 距離都很大（因為都偏離 μ_A）
→ 影像 B 全圖被當成異常 → 完全失效
```

### 理論上也可以「pool across scenes」訓共用 VAE

這條路存在，但有明確缺點：

| 缺點 | 說明 |
|------|------|
| 學到的是「各影像水的平均樣子」| 對任何單張影像都不夠精確，邊緣水體容易漏判 |
| 訓練量過大 | 100 張影像 × 500 萬像素 = 5 億像素，訓練時間不可接受 |
| 不同影像的尺度差異 | 需要額外做 per-scene normalization，否則高霾影像的像素會主導訓練 |
| 新場景需重訓 | 加入新影像仍需重訓，失去「共用」的靈活性 |

### 結論

Per-scene 訓練是最合理的取捨：訓練成本可控（500 萬像素 × 50 epochs ≈ 2-3 分鐘/場景），且對每張影像的特定光譜環境都能精確建模。

## 11. v3 濫抓的 4 個根本原因

> 本節源自 2026-06-18 Q&A，使用者觀察到「v3 sliding window 是不是只要 patch 內有一點異常就誤判？」——這個直覺對了一半，實際有 4 個層次的問題。

### 4 個根本原因

**原因 ①：VAE 訓練資料沒篩乾淨**

```
v3 訓練流程（有問題的）：
    全部 3,000 個 patch，不分好壞，直接取所有像素訓 VAE
    → VAE 學到的「正常」包含：乾淨水 + 雲邊緣 + 海岸線 + 淺水
    → 異常標準被拉寬（這些都學成「正常」了）
    → 真正的油汙要更「奇怪」才能被偵測到

v9 修正：patch_is_scoreable 嚴格篩選，只有確定是乾淨水體的 patch 才進訓練集
```

**原因 ②：Patch 聚合對少數異常像素過度敏感**

```
5th percentile 聚合的數學含義：
    一個 256×256 patch = 65,536 個像素
    5th percentile = 排序後第 3,277 個（從高分端數）
    → 只要 patch 內有 ≥ 5% 的像素異常分數高，整個 patch 就被拉高

具體例子：
    patch 內容：95% 乾淨水（分數低）+ 5% 雲邊緣（分數高）
    → 5th percentile 抓到的是雲邊緣的分數
    → 整個 patch 被誤判為異常
    → 即使「大部分是水」，只要有一小塊非水區域就中招
```

**原因 ③：EM 門檻可能設太鬆**

```
EM 的前提假設：patch 分數分布是 2 個高斯的混合
    正確情境：正常 patch 一堆（分數低）+ 異常 patch 一堆（分數高）
              → EM 找到兩群的交點，門檻合理

v3 的實際情境：分布可能是「1 個高斯 + 長尾」
    ─ 因為 VAE 沒篩乾淨，正常 patch 的分數本來就偏高且散
    ─ 「高分群」實際上大多是訓練不佳造成的偽高分
    → EM 強行擬合成 2 高斯，把長尾當成第 2 個高斯
    → 門檻被拉得比真正合理的值低
    → 大量本應是正常的 patch 被歸入「異常」群
```

**原因 ④：沒有油汙物理驗證**

```
v3 的輸出邏輯：
    通過 EM 門檻的 patch → 全部列為油汙候選

問題：通過 EM 門檻的可能是：
    雲邊緣（光譜異常，但不是油）
    海岸線（水陸交界，反射率混合）
    淺水區（底部反射讓 SWIR 升高）
    小船（點目標，周圍 patch 被拉高）
    ← 還有油汙（我們真正想要的）

v9 修正：NDOI z-score ≥ 0.5 作為油汙物理驗證，排除非油異常
```

### v3 問題 → v9 解法對應表

| v3 問題 | v3 後果 | v9 解法 |
|---------|---------|---------|
| ① VAE 訓練資料不純（含雲/陸/淺水）| 異常標準寬鬆，分數整體偏高 | `patch_is_scoreable` 嚴格篩選訓練像素 |
| ② 5th percentile 對少數高分像素過敏 | patch 內有 5% 異常就整體被拉高 | 維持 5th percentile，但依賴前置篩選降低偽高分 |
| ③ EM 在非雙峰分布時門檻設太低 | 長尾被誤認為異常群，大量 FP | 間接改善（訓練資料純化讓分布更接近雙峰）|
| ④ 無物理後過濾 | 雲/陸/淺水通通被列為油汙候選 | NDOI z-score ≥ 0.5 驗證光譜特性 |

### v9 還有 FP 的根本原因

v9 改善了 ① ③ ④，但 **② 5th percentile 過敏** 這個根本問題還沒解決：

```
只要 patch 內有 ≥ 5% 的像素是「非乾淨水體」（雲邊緣、海岸、glint），
整個 patch 的異常分數就會被拉高，通過 EM 門檻後再被 NDOI 過濾，
但如果那 5% 的非水像素剛好也有油汙光譜特徵（SWIR 略高），
NDOI z-score 也過不了關 → 整 patch 被誤判為油汙
```

### 未來改進方向

| 方向 | 說明 | 優點 | 缺點 |
|------|------|------|------|
| 改 median 聚合 | 每 patch 取分數中位數 | 對少數高分像素免疫，FP 大幅降低 | 小油汙（< 50% patch 覆蓋）可能被壓低，Recall 下降 |
| 改 mean 聚合 | 每 patch 取平均分數 | 保守，穩定 | 對小油汙非常不敏感 |
| mean + percentile 雙重門檻 | 兩者同時高才算 | 兼顧 | 門檻參數增加，需調參 |
| Patch 級 spatial coherence | 鄰居 patch 也異常才算 | 排除孤立 FP | 會錯過孤立小油汙 |

## 12. 給投影片報告用的 3 個重點 Takeaways

> 本節精煉自 2026-06-17~2026-06-18 的完整 Q&A，適合作為投影片的「結論/貢獻/討論」頁使用。

### Takeaway 1：VAE 訓練策略——per-scene，不跨影像共用

> **VAE 在每張影像上獨立訓練（不跨影像共用）。**

```
訓練資料來源：
    該影像所有通過篩選（patch_is_scoreable）的 patches 的水體像素
    → 池化成約 500 萬像素
    → 這些是「這張影像上，我確定是乾淨水體的像素」

不是每 patch 訓一個（計算量 3,000×，不可行）
不是所有影像共用一個（跨影像光譜差異太大，會失效）
而是每張影像訓一個，訓練資料是該影像的水體像素池
```

**投影片重點**：這個設計讓 VAE 學到的是「這張影像、這個時刻、這個大氣狀況下的正常水體」，而不是一個泛化的「所有水體」——這是能在 Recall=0.37 上做出成績的關鍵設計決策。

### Takeaway 2：論文 AUC 0.99 ≠ 我的 Recall 0.37，差別在場景複雜度與評估指標

> **這不是 VAE+RX 不夠強，是這個架構天生不適合 multi-anomaly 場景。**

```
論文場景：                    我的場景：
  正常：水泥地 + 跑道            正常：深海水（多種狀況）
  異常：飛機（1 種）              想要的異常：油汙（1 種）
                                  不想要的異常：雲、陸、船、glint（5 種）

評估指標：
  論文：AUC（只要飛機分數 > 背景就行）
  我的：Recall（要求油汙分數 > 所有其他異常）
  ↑ 後者困難得多
```

**投影片重點**：VAE+RX 能回答「哪裡不像背景？」，但無法回答「哪裡不像背景，而且是油汙？」。在 multi-anomaly 場景下，後半句的驗證（NDOI z-score）是必要的，但 NDOI 本身也有極限。

### Takeaway 3：v9 未解決的問題——5th percentile 對少數像素過度敏感

> **v9 濫抓的根本原因還在 patch 聚合方式，不完全是訓練資料問題。**

```
問題核心：
    一個 256×256 patch 有 65,536 個像素
    5th percentile = 最異常的 3,277 個像素決定整個 patch 的分數
    
後果：
    patch 內 95% 乾淨水 + 5% 雲邊緣
    → 5th percentile 抓到雲邊緣分數
    → 整 patch 被列為異常候選
    → 即便 NDOI 過濾，若雲邊緣光譜恰好偏高 SWIR，還是誤判

即使做到：
    ✅ patch_is_scoreable 嚴格篩訓練資料
    ✅ NDOI z-score ≥ 0.5 物理驗證
    → 只要 patch 內有 ≥5% 是雲邊緣/海岸/glint，整 patch 還是可能誤判
```

**投影片重點**：要根治這個問題，需要改變 patch 聚合方式（median 或 spatial coherence）。這是 v9 → v10 的最高優先改進方向。
