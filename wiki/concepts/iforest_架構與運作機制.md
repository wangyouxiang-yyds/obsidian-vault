## 1. iForest 的本質：是 model，但既不是統計模型也不是深度學習

### 三種「模型」的對照（純概念釐清，不做比較）

iForest 屬於 **「樹結構 ensemble」** 這一類 model：

| 面向 | iForest |
|------|---------|
| 學什麼？ | 一堆**隨機切分的二元樹** |
| 怎麼學？ | 隨機選特徵 + 隨機選閾值 |
| 存什麼？ | 100~800 棵樹的結構（節點＋分支）|
| 有 weight 嗎？ | ❌ 沒有 |
| 有機率分布參數嗎？ | ❌ 沒有 |
| 有 loss function 嗎？ | ❌ 沒有 |
| 需要 label 嗎？ | ❌ 不需要 |
| 是 model 嗎？ | ✅ 是 — fit() 後可以存 / 載入 |

### iForest 的核心洞見

不是統計、不是學習，是 **「異常點容易被孤立」這個幾何/結構事實**：

- 你隨便切空間 → 異常點周圍稀疏 → 幾刀就能單獨切出來
- 正常點周圍密集 → 要很多刀才能孤立

→ **「路徑越短 = 越異常」**

## 2. 「樹」是什麼？「建樹」是什麼意思？

### 樹的基本概念

樹（tree）是一種**樹狀資料結構**，由節點與分支組成。iForest 用的是**二元樹（iTree）**——每個節點最多 2 個子節點。

```
            根節點 (root)
           /          \
         節點          節點
        /    \        /    \
      葉    葉      葉     葉
```

**每個內部節點**記錄兩件事：
- 用哪個**特徵**來切
- 用什麼**閾值**來切

**每個葉節點**就是一個資料點（或一小群點）。

### 用 8 個像素的迷你例子看「建樹」

假設只有 8 個像素，每個像素 2 個特徵（B02 藍光、B11 SWIR1）：

```
像素編號    B02     B11
  P1       0.10    0.05    ← 海水
  P2       0.12    0.06    ← 海水
  P3       0.11    0.04    ← 海水
  P4       0.13    0.07    ← 海水
  P5       0.09    0.05    ← 海水
  P6       0.10    0.04    ← 海水
  P7       0.10    0.06    ← 海水
  P8       0.45    0.30    ← 油汙（明顯不一樣）
```

**建樹步驟：**

1. **隨機選一個特徵**：擲骰子 → 選到「B02」
2. **在 B02 的 min~max 之間隨機選一個切點**：
   - min=0.09, max=0.45 → 隨機選到 **0.20**
3. **切成兩堆**：
   ```
   B02 < 0.20？
     ✅ 是 → 左邊：[P1,P2,P3,P4,P5,P6,P7] （7 個海水）
     ❌ 否 → 右邊：[P8]                      （1 個油汙，孤立完成）
   ```
4. **對左邊 7 個遞迴**：隨機再選 B11，隨機切點 0.055...
5. **直到每個葉節點只剩 1 個像素**

**最終樹的樣子：**

```
              [根: B02 < 0.20?]
             /                \
       [B11<0.055?]            P8  ← 路徑長度=1（第一刀就孤立！）
       /        \
   [B02<0.105?] [B02<0.115?]
    /    \        /     \
   P5    [...]  P4    [B11<0.065?]
         /  \             /    \
       P6   P1          P2    P7
```

### 為什麼這樣可以找出異常？

| 像素 | 從根到葉的路徑長度 |
|------|------------------|
| **P8（油汙）** | **1** ← 第一刀就被孤立 |
| P5（海水） | 3 |
| P4（海水） | 3 |
| P6, P1 | 4 |
| P2, P7 | 5 |

- 油汙 P8 在特徵空間離海水很遠 → 隨便一刀就切到它一個 → 路徑超短
- 海水 P1~P7 擠在一起 → 要切很多刀才能分開 → 路徑長

**核心結論：路徑越短 = 越異常**

## 3. 為什麼要建 800 棵樹？

每棵樹「隨機選特徵、隨機選閾值」—— **單棵樹的結果可能運氣不好**：

- 某棵樹剛好先切 B11、又選到爛閾值 → 油汙可能要切 3 刀才孤立
- 但 800 棵樹平均下來，**異常點的平均路徑長度一定比正常點短**

這就是「隨機 + 平均」的力量——單棵不準，800 棵的平均很穩定。

Duan 2022 在 GM01 上做 sensitivity analysis（T ∈ {200, 400, ..., 1600}），發現 **T=800** 是 AUC 峰值。

## 4. Duan 2022 對 GM01 的完整處理流程

GM01 規格：1200×633，224 bands，AVIRIS 機載高光譜，7.6m 解析度。

### Step 1：原始資料

```
GM01 shape = (1200, 633, 224)
每個像素 = 一個 224 維向量（在 224 個波段上的反射率）
總像素數 = 1200 × 633 = 759,600
```

### Step 2：雜訊波段移除（224 → 約 180）

對每個波段用 Gaussian-Laplacian 公式估雜訊變異數 σ_n：

```
門檻 = (1 / 2I_N) × Σ σ_n
σ_n < 門檻 → 保留
σ_n ≥ 門檻 → 丟掉
```

### Step 3：KPCA 降維（180 → 25）

KPCA = Kernel PCA，RBF kernel，壓到 **D=25** 維。
論文 §4.1 sensitivity analysis 確認 D=25 最佳。

### Step 4：攤平成 2D 矩陣

```
(1200, 633, 25)
    ↓ 攤平
(759600, 25)  ← 每行是一個像素，25 維特徵
```

### Step 5：建 iForest（800 棵 iTree）

```
for t = 1 to 800:
    Step 5.1：從 759,600 個像素中隨機抽 K=256 個
              （注意：抽到的 256 個不一定包含油汙）
    Step 5.2：用這 256 個 25 維向量建一棵 iTree
              - 隨機選 25 維中的某一維
              - 隨機切點
              - 遞迴切到葉節點只剩 1 個
              - 樹高上限 = log₂(256) = 8
    Step 5.3：把這棵樹存起來

→ 最後得到 800 棵 iTree
```

**關鍵理解**：每棵樹只看 256 個像素，但建好之後是「對 25 維特徵空間做了一套隨機切分規則」，可以套用到**任何 25 維向量**上。

### Step 6：對全部 759,600 個像素打分

```
for 每個像素 x in 759,600 個像素:
    路徑長度集合 = []
    for t = 1 to 800:
        從 iTree_t 的根節點開始
        依照節點規則一路走到葉節點
        記錄路徑長度 h_t(x)
    
    avg_path = mean(路徑長度集合)
    p(x) = 2^( -avg_path / c(256) )
    # c(m) = 2H(m-1) - 2(m-1)/m, H 是 harmonic number

→ 得到 probability map (1200, 633)，每個像素 ∈ (0, 1]
  越接近 1 = 越異常（油汙）
  越接近 0 = 越正常（海水）
```

### Step 7：k-means 二分

```
probability_map → 攤平成 759,600 個分數
k-means(n_clusters=2):
    群 A：分數高的 → pseudo label 油汙（Lo）
    群 B：分數低的 → pseudo label 海水（Ls）
```

### Step 8：SVM 二次分類（用 pseudo label 訓練）

```
從 Lo 抽 1%、Ls 抽 1% 當訓練集
SVM(Gaussian kernel) + 5-fold CV 調參
重新對全部像素分類 → 初始 detection map O
```

**為什麼要 SVM 二次分類？** iForest 的 probability map 通常 noisy，SVM 用更平滑的決策邊界把雜訊磨掉。

### Step 9：ERW 後處理

```
ERW = Extended Random Walker（類似 CRF）
參數：γ = 10⁻⁵, β = 710
輸入：O + 空間鄰近資訊
輸出：refined map

論文 GM01 實測：
  不做 ERW：AUC = 0.9166
  做了 ERW：AUC = 0.9472  ← 提升 3%
```

### Step 10：取 argmax 得最終結果

```
最終 ĉ ∈ {0, 1}^(1200×633)
GM01 AUC = 0.9472（論文 Table 2）
```

### 完整資料流大綱

```
GM01 (1200,633,224)
    ↓ 雜訊波段移除
(1200,633,180)
    ↓ KPCA D=25
(1200,633,25)
    ↓ 攤平
(759600, 25)
    ↓ ── iForest (T=800, 每棵 K=256) ──→ 800 棵 iTree
    ↓ 對每個像素打分（走 800 棵樹）
probability_map (1200,633) ∈ (0,1]
    ↓ k-means 二分
pseudo label (1200,633) ∈ {0,1}
    ↓ 抽 1% 訓 SVM
SVM detection map O (1200,633)
    ↓ ERW 後處理
最終結果 ĉ (1200,633)
```

## 5. 訓練資料和測試資料是同一張影像，這樣不會有問題嗎？

### 是的，確實同源

| 階段 | 用什麼資料？ |
|------|-------------|
| 建 iForest（"訓練"）| GM01 隨機抽 256×800 個像素（從 759,600 中抽）|
| 打分（"測試"）| GM01 全部 759,600 個像素 |

**訓練資料 ⊂ 測試資料**，這在監督式 ML 裡叫**資料外洩**。

### 但這在 iForest 不算外洩 — 因為它從頭到尾沒看 label

**監督式 ML 的外洩問題**：
```
有 label：(x_1, y_1=油汙), (x_2, y_2=海水), ...
    ↓ 訓練（model 記住 x → y 對應）
測試時拿同一批 x → 當然準（model 偷看過答案）
```

**iForest**：
```
無 label：x_1, x_2, ..., x_n（沒人告訴它誰是油汙）
    ↓ 建樹（純粹隨機切空間）
打分時：對每個 x 算路徑長度 → 異常分數
↑ iForest 從頭到尾沒看過正確答案，沒有偷看的可能
```

iForest **學的不是「x → y 對應」，是「這批資料的空間結構長什麼樣」**。它在訓練階段甚至不知道「油汙」這個概念存在。

### 觀念整理：iForest 在做的是 transductive analysis，不是 inductive learning

|  | 監督式 ML | iForest |
|--|----------|---------|
| 目標 | 預測新資料的 label | 在這批資料裡找異常點 |
| 「答案」在哪？| 由訓練 label 給定 | 由資料分布的幾何結構決定 |
| train/test 分開重要嗎？| **非常重要** | **不重要** |
| 學術領域 | Inductive learning（歸納）| Transductive analysis（直推）|

**iForest 本來就不適用 train/test split 的概念**。它做的是「對一個固定資料集找異常」，不是「學個 model 預測未來」。

## 6. iForest 真正會失敗的情境：Contamination（污染）

> 真正會讓 iForest 出問題的不是「同源」，而是 **異常點佔比太高**。

### Contamination 定義

iForest 假設「異常點少」（few）。如果：

- 油汙佔比 **< 5%** → 抽 256 個像素時大部分是海水 → 學到的「正常分布」乾淨 → 油汙易被孤立 ✅
- 油汙佔比 **> 30%** → 抽 256 個像素時夾雜大量油汙 → 樹會以為油汙也是「正常」 → 油汙不易被孤立 ❌

### 對 GM01 的影響

GM01 油汙佔比約 4%，剛好落在 iForest 適用範圍 → AUC 0.94 沒問題。

如果油汙佔了 40%，iForest 就會「習慣」油汙，把它當正常看，可能失敗。

## 7. 對本專案（v9 DeepRX pipeline）的啟示

雖然本專案用 VAE+RX 而非 iForest，但「同源訓測」的邏輯一樣，理解 iForest 的這套觀念可以類推：

| 真正要擔心的問題 | 對應 v9 的解法 |
|----------------|---------------|
| 影像中**陸地+雲太多** → VAE 把陸地/雲學成「正常」 → 油汙混在裡面找不到 | v9 用 `patch_is_scoreable` 嚴格篩水體 |
| 影像中**油汙佔比太高** → VAE 把油汙學成「正常」 | 場景油汙都 < 1%，不會發生 |

v9 強化「先排陸地+雲」正好就是在避免**真正會出問題**的 contamination，不是處理「資料外洩」這個假議題。

---

## 一句話總結

> **iForest 是一個 model，但這個 model 的內容就是「隨機建出來的 100~800 棵二元樹」。它沒有 weight、沒有機率分布、沒有 loss function、不需要 label。建樹是「隨機抽 256 個像素 + 隨機切空間」；打分是「全部像素走過所有樹，看平均路徑長度」。訓練/測試同源不是問題，因為它沒看過答案；真正會失敗的是 contamination —— 異常點佔比太高。**
