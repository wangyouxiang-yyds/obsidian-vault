# Gemini CLI 的角色：雜工，不是第三個 harness

Gemini 與 Claude Code / Codex 地位不對等。Gemini 不參與架構判斷、方法論選擇、除錯根因判斷，
只執行「範圍已鎖死、不需要判斷力」的低價值任務。

## 可以交給 Gemini 的工作（範例）
- 格式轉換（例如 CSV → Markdown 表格、批次改副檔名）
- 純機械式批次改檔（例如把 100 個檔名裡的某個固定字串取代掉）
- 已經寫好內容的簡單摘要（不需要判斷取捨，例如把一份 log 摘要成三行重點）

## 不能交給 Gemini 的工作
- 任何需要判斷「這個異常是根源還是症狀」的除錯任務（例如 Branch A/B 的偽陽性/過擬合診斷）
- 任何會寫入 `.agents/` 正本、`CLAUDE.md`、`AGENTS.md` 的任務
- 任何需要在多個方案間做選擇判斷的任務（例如評估新方法路線可行性）

## 交辦格式
一句話講清楚：目標、輸入檔案、預期輸出格式、驗收條件（例如「輸出表格要有 N 列」）。
Gemini 的輸出一律視為未驗證：Claude Code 或 Codex 要做一次 read-back 確認後才能採用，不可直接信任、不可直接轉呈使用者。
